# PWA-2: Архив статей

PWA-сайт для чтения архива статей из `txt`-файлов с таблицей, сортировкой, поиском и статусами.

## Что реализовано

- Таблица из 4 колонок: `Категория`, `Автор`, `Название`, `Статус`.
- Источник данных: первые 3 строки каждого файла `.txt`:
  - `Категория: ...`
  - `Автор: ...`
  - `Название: ...`
- Если в категории несколько значений через запятую, одна статья показывается в таблице несколько раз (по каждой категории).
- Сортировка по клику на заголовки колонок (включая `Статус`).
- Поиск по любому слову из полей `Категория`, `Автор`, `Название`.
- Клик по `Название` открывает полный текст статьи в модальном окне.
- Статус статьи в 3 состояниях:
  - `Без статуса`
  - `Приоритетный`
  - `Выполненный`
- Статус хранится в `localStorage` и применяется сразу ко всем строкам одной статьи.
- PWA: `manifest.webmanifest` + `service worker` для офлайн-кэша.

## Структура

- `txt/` — локальный архив статей (`.txt`, cp1251/ANSI, LF).
- `scripts/build-dataset.mjs` — конвертация архива в `public/data/articles.json`.
- `src/` — фронтенд.
- `public/` — статические PWA-ресурсы.

## Запуск

```bash
npm install
npm run build:data
npm run dev
```

После этого откройте адрес из консоли Vite.

## Сборка для продакшна

```bash
npm run build:data
npm run build
```

Готовый сайт будет в папке `dist`.

## Ограничение по репозиторию и оптимальное решение

В репозиторий нельзя грузить `txt`-архив, поэтому:

1. Храните `txt/` только локально (оно уже в `.gitignore`).
2. На локальной машине запускайте `npm run build:data` для генерации `public/data/articles.json`.
3. Для публикации сайта:
   - либо загружайте `dist/` + `articles.json` в облачное хранилище (Google Cloud Storage, S3, Cloudflare R2),
   - либо храните `articles.json` в отдельном приватном/служебном хранилище и отдавайте по URL.
4. Приложение можно легко переключить на удаленный URL данных (изменить путь в `src/main.js`).

Почему это оптимально:

- Исходные `txt` не попадают в Git.
- Получается один готовый датасет для быстрого чтения в браузере.
- Простая публикация как статического PWA без серверной логики.

## Команды

- `npm run build:data` — пересобрать базу данных из `txt/`.
- `npm run dev` — локальный запуск.
- `npm run build` — production-сборка.
- `npm run preview` — локальный просмотр production-сборки.
